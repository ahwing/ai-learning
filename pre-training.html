训练动态：Loss 曲线与 Scaling Law

预训练的核心目标是让 loss 持续下降，Scaling Law 指导资源分配

📉 典型预训练 Loss 曲线
🌍 现实案例：OpenAI 的 Kaplan 论文 (2020) 首次揭示了模型性能与参数量、数据量、计算量之间的幂律关系，从此 LLM 竞赛从"改架构"变成了"堆资源"。
Loss
Training Steps →
● 初始阶段：loss 快速下降
● 中期：稳定下降
● 后期：收敛变慢
📐 Chinchilla Scaling Law
核心公式
L(N, D) = A/Nα + B/Dβ + E
N = 模型参数量
D = 训练数据量 (tokens)
最优比例：D ≈ 20 × N
实际案例
• LLaMA-3 8B → 15T tokens (1875×)
• LLaMA-3 70B → 15T tokens (214×)
• DeepSeek-V3 → 14.8T tokens
现代趋势：过度训练小模型（超过 Chinchilla 最优比例），换取推理时更低成本

核心概念深入

用直觉理解预训练背后的"为什么"

🎯 为什么"预测下一个词"能让模型变聪明？

类比：想象你在看一本推理小说，每读完一段就猜下一段会写什么。要猜得准，你必须理解人物关系、故事逻辑、写作风格——你被迫学会了"理解"。

核心直觉：预测下一个 token 看似简单，但要做好它，模型必须隐式学会：

语法规则
"他正在" → 动词
世界知识
"水的沸点是" → 100
逻辑推理
"如果A>B且B>C，则A" → >C
情感理解
"他失去了一切，感到" → 绝望

💡 本质：Next-token prediction 是一个"压缩"任务。要用有限参数预测任意文本的下一个词，模型被迫将人类知识压缩进权重中。压缩得越好 = 理解得越深。

📊 交叉熵损失：直觉解释

类比：想象你是天气预报员。你说"明天 90% 会下雨"，结果真的下了——你的预测很好（低损失）。如果你说"10% 会下雨"结果下了——你的预测很差（高损失）。交叉熵就是衡量"你有多惊讶"的指标。

Loss = -log(Pmodel(正确答案))
模型说 P=0.9
Loss = 0.10
很确定且对了 ✓
模型说 P=0.5
Loss = 0.69
不太确定
模型说 P=0.01
Loss = 4.60
完全猜错 ✗

💡 为什么用交叉熵而不是"对/错"？因为它是连续可微的——梯度能告诉模型"往哪个方向调整"以及"调多少"。二元对错无法提供这种精细信号。

⚡ 为什么用 AdamW？优化器对比

类比：想象你在浓雾中下山。SGD 是"只看脚下坡度走"；Adam 是"记住之前走过的方向（动量）+ 根据地形陡峭程度调整步幅（自适应学习率）"；AdamW 额外加了"别走太远离山顶"的约束（权重衰减）。

优化器 优点 缺点 适用场景
SGD+Momentum 内存少，泛化好 收敛慢，需精调 lr CV 任务、小模型
Adam 自适应 lr，收敛快 权重衰减实现有 bug 通用深度学习
AdamW ✓ 修正权重衰减，稳定 内存占用大（2x 状态） LLM 预训练标配

💡 Adam vs AdamW 的关键区别：Adam 把权重衰减混在梯度里（L2 正则），导致自适应学习率"稀释"了衰减效果。AdamW 将权重衰减解耦出来，直接作用于参数，正则化效果更纯粹。

📈 学习率调度的直觉

类比：学开车。Warmup = 刚上路慢慢加速（防止猛踩油门失控）；Peak LR = 高速公路上全速前进（快速学习）；Cosine Decay = 接近目的地逐渐减速（精细调整，别冲过头）。

🔥 Warmup 阶段
• 为什么？初始参数随机，梯度方向不可靠
• 大学习率 + 随机梯度 = 训练爆炸
• 先用小 lr 让 Adam 的动量估计"热身"
• 通常 1000-2000 步
🚀 Peak → Decay
• 为什么余弦而非线性？
• 余弦前期衰减慢 → 充分探索
• 后期衰减快 → 快速收敛
• 比线性衰减多出 ~5% 性能
🎯 Min LR
• 通常为 peak 的 1/10
• 不降到 0：保持微小学习能力
• 为后续继续训练留余地
• 典型值：3e-5

替代方案与优化

Next-token prediction 不是唯一选择，了解其他路线和前沿优化

🔄 替代训练目标
Masked Language Modeling (MLM)
代表：BERT、RoBERTa
做法：随机遮住 15% 的词，让模型猜
✅ 双向上下文理解强
❌ 不能直接生成文本
Denoising / Span Corruption
代表：T5、UL2
做法：删除/打乱文本片段，让模型恢复
✅ 灵活，可混合多种目标
❌ 训练效率低于 CLM
Prefix LM / Fill-in-Middle
代表：GLM、CodeLLaMA FIM
做法：给前缀和后缀，预测中间内容
✅ 适合代码补全场景
❌ 增加训练复杂度
Multi-Token Prediction
代表：Meta MTP (2024)
做法：同时预测未来 N 个 token
✅ 更好的表征，推理可加速
❌ 训练内存增加
🧪 新一代优化器
AdamW 是当前标配，但研究者在探索更高效的替代方案：
Lion (Google, 2023)
只用 sign(momentum) 更新，极简
✅ 内存减半（无需二阶矩）
✅ 某些任务超越 AdamW
❌ 对超参数更敏感
Sophia (Stanford, 2023)
用 Hessian 对角近似做预条件
✅ 收敛速度快 2x
✅ 自动裁剪大梯度
❌ Hessian 估计有额外开销
Adafactor (Google)
分解二阶矩为行/列统计量
✅ 内存大幅减少
✅ T5/PaLM 使用
❌ 训练不如 AdamW 稳定
📚 数据配比与课程学习

核心问题：不同来源的数据按什么比例混合？顺序重要吗？

数据配比策略
• 静态配比 — 预设固定比例（如 Web:60%, Code:20%, Book:10%, Academic:10%）
• 动态调整 — 根据各领域 loss 下降速度动态调权（DoReMi, 2023）
• 数据退火 — 训练末期提高高质量数据比例（LLaMA-3 策略）
课程学习 (Curriculum Learning)
• 从易到难 — 先学简单短文本，再学复杂长文档
• 上下文长度递增 — 4K → 32K → 128K 分阶段扩展
• 质量递增 — 先用大量普通数据建基础，后用精选数据精炼
🛡️ 训练稳定性技术

大模型训练动辄数月、花费千万美元，一次 loss spike 可能浪费数天算力。稳定性至关重要：

梯度裁剪 (Gradient Clipping)
当梯度范数超过阈值（通常 1.0），等比缩小
类比：给汽车装限速器，防止失控
几乎所有大模型训练都使用
Loss Spike 处理
检测到 loss 突然飙升时：
1. 跳过当前 batch
2. 回滚到最近 checkpoint
3. 降低学习率重新训练该段数据
LLaMA-3 报告了约 10 次 loss spike
Checkpoint 平均
取训练后期多个 checkpoint 的参数平均值
类比：多次考试取平均分，减少偶然波动
通常提升 0.1-0.3% 性能
其他技巧
• z-loss 正则化（稳定 logits 范围）
• 嵌入层梯度缩放
• QK-Norm（注意力数值稳定）
• μP 参数化（超参数迁移）

常见问题

关于预训练最常被问到的问题

Q1：预训练数据用完了怎么办？数据会不会不够用？

这是真实存在的问题（"数据墙"）。应对策略包括：

• 合成数据生成（用已有模型生成训练数据，如 Phi 系列）
• 多 epoch 训练 + 正则化（重复使用数据但防止过拟合）
• 多模态数据引入（图片、视频、音频转文本描述）
• 提高数据质量而非数量（精选数据效果可能更好）
Q2：预训练出来的模型为什么不能直接用？还需要 SFT/RLHF？

预训练模型是"博学但不听话的学者"。它学会了语言和知识，但：

• 不会遵循指令（你问问题，它可能继续写文章而非回答）
• 没有对齐（可能输出有害、偏见内容）
• 格式不可控（不知道何时停止、如何组织回答）

SFT 教它"如何回答"，RLHF 教它"什么是好的回答"。三者缺一不可。

Q3：为什么不用更大的 batch size？越大越好吗？

不是越大越好。Batch size 有"甜蜜点"：

• 太小：梯度噪声大，训练不稳定，GPU 利用率低
• 太大：泛化能力下降（"sharp minima"问题），且收益递减
• 最优 batch size 与学习率相关（线性缩放规则）

实践中常用"batch size ramp-up"：从小 batch 开始，逐步增大到目标值。这兼顾了早期探索和后期稳定。

Q4：训练中途发现数据有问题（如有毒数据），能补救吗？

可以，但代价不同：

• 最佳方案：从最近的干净 checkpoint 重新训练（回滚）
• 折中方案：继续训练但加入"遗忘"数据（对冲有毒数据的影响）
• 后期补救：通过 SFT/RLHF 阶段纠正（效果有限）

这就是为什么数据清洗如此重要——预防远比治疗便宜。回滚一周的训练可能浪费数百万美元。

Q5：小团队/个人还有可能做预训练吗？

取决于规模目标：

• 1-3B 模型：8×A100 约 1-2 周，成本 $5K-$20K，个人可行
• 7-13B 模型：64×A100 约 2-4 周，成本 $50K-$200K，小团队可行
• 70B+ 模型：1000+ GPU 数月，成本 $2M+，需要大公司资源

替代路线：在开源基座模型上做继续预训练（Continual Pre-training），用 1/100 的成本获得领域专精能力。
