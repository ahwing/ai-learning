Amazon Bedrock 全景 + 选型决策

一站式托管 AI 平台 — 从模型调用到企业级 AI 应用的完整路径

🎯 核心定位：Bedrock 是 AWS 的"模型即服务"平台 — 无需管理基础设施，通过 API 调用顶级 FM，加上 RAG、Agent、Guardrails 等企业功能，快速构建生产级 AI 应用。

🌳 架构决策树（交互式）

点击每个问题，展开推荐方案

❓ 需要自定义模型吗？（模型行为不满足需求）

✅ 是 → Fine-tuning / Continued Pre-training

领域特定术语、特定输出格式、品牌语调定制。Bedrock 支持对 Titan、Llama、Cohere 等模型微调。

适用：垂直行业、特定格式要求

❌ 否 → 直接 API 调用 + Prompt 工程

大多数场景下，好的 Prompt 设计就够了。先试 Prompt Engineering，不行再考虑 Fine-tuning。

适用：通用任务、快速上线
❓ 需要最新/企业私有知识吗？

✅ 是 → Knowledge Bases (RAG)

连接 S3、Confluence、SharePoint 等数据源，自动向量化 + 检索增强。支持多种 chunking 策略和混合搜索。

适用：企业知识库、文档问答

❌ 否 → 直接 Prompt / In-context Learning

如果知识已在模型训练数据中，或可以在 Prompt 中提供足够上下文，无需 RAG。

适用：通用知识、短上下文
❓ 需要执行动作/调用外部系统吗？

✅ 是 → Bedrock Agents

Agent 可以调用 Lambda 函数、查询数据库、调 API。支持 ReAct 循环、多步规划，自动编排工具链。

适用：自动化工作流、多步任务

❌ 否 → 单次推理 (InvokeModel)

简单的文本生成、摘要、翻译等不需要外部交互的任务，直接调 InvokeModel API 即可。

适用：文本生成、分类、翻译
❓ 对延迟和成本的要求是什么？

⚡ 低延迟高吞吐 → Provisioned Throughput

预留模型算力，保证稳定低延迟。适合生产环境高并发场景。按小时计费，承诺期越长折扣越大。

成本：高（固定费用）

💰 弹性按量 → On-Demand

按 token 计费，无预付。适合流量不稳定或开发测试阶段。

成本：中（按使用量）

🐢 不急 → Batch Inference

批量处理离线任务，最高可省 50%。适合大规模数据处理、不要求实时响应的场景。

成本：低（批量折扣）

💰 成本优化策略

四大策略降低 Bedrock 使用成本

⚡
Provisioned Throughput
预留算力，保证稳定 TPS
适用：生产环境高并发
• 1/6 月承诺期折扣更大
• 自定义模型必须用此模式
• 可预测的固定月费
🌍
Cross-Region Inference
自动路由到有空闲容量的区域
适用：高并发/容量受限
• 自动负载均衡
• 提升可用性和吞吐
• 无需代码更改
📦
Prompt Caching
缓存重复的长 system prompt
适用：System Prompt 很长且复杂
• 减少重复 token 计费
• 降低 90% 缓存命中部分成本
• TTFB 显著降低
🐢
Batch Inference
离线批量处理，最高 50% 折扣
适用：大规模离线任务
• 文件输入/输出（S3）
• 不要求实时响应
• 数据标注、批量翻译等
策略 延迟 成本 适用流量模式 承诺要求
On-Demand ✅ 正常 基准价 波动/低量 无
Provisioned ✅ 最低 固定月费 稳定高并发 1-6个月
Batch ⏱ 分钟~小时 节省 50% 离线大批量 无
Prompt Caching ✅ 更低 节省 ~90% (cached) 重复长 Prompt 无
Cross-Region ✅ 正常 同价 高并发/跨区 无

⚖️ Bedrock vs SageMaker — 如何选择？

两者互补，而非替代关系

🟠 Amazon Bedrock

✅ 无需管理基础设施
✅ 多家顶级 FM 即开即用
✅ 内置 RAG、Agent、Guardrails
✅ Serverless，按 token 计费
✅ 快速原型 → 生产
⚠️ 模型选择有限（仅合作伙伴）
⚠️ 自定义训练能力有限
选 Bedrock：快速用 FM 构建应用，不想碰 infra
VS

🔵 Amazon SageMaker

✅ 完全控制训练和部署
✅ 支持任何开源/自研模型
✅ 自定义训练 pipeline
✅ GPU 实例精细控制
✅ MLOps 全流程
⚠️ 需要 ML 工程团队
⚠️ 需要管理基础设施
选 SageMaker：自研模型、完全控制训练、需要 MLOps
🤝 组合使用：很多企业两者结合 — 用 SageMaker 训练/微调自研模型，用 Bedrock 调用 Claude/Nova 做推理；或用 SageMaker 做 embedding 模型，Bedrock 做生成模型。
维度 Bedrock SageMaker
目标用户应用开发者ML 工程师/科学家
模型来源合作伙伴 FM任意模型
基础设施全托管 Serverless自管 GPU 实例
训练能力有限微调完全自定义
部署复杂度极低（API 调用）中-高
成本模型按 token/请求按实例小时
上线速度分钟级天-周级
